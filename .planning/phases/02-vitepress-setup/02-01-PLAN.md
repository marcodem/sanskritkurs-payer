# Phase 2 Plan: VitePress Setup & Erster Build

This plan focuses on finalizing the VitePress platform setup by implementing automatic lesson numbering, establishing the grammar topics index, and ensuring asset localization.

## Path to Success

### Wave 1: Core Platform Enhancements
- [ ] Task 1: **Implement Automatic Numbering via CSS**. Update `custom.css` to use CSS counters for `h2` and `h3` tags within the document area.
- [ ] Task 2: **Create Grammar Topics Index**. Initialize `docs/grammatik.md` with a structural template to act as a central conceptual entry point.

### Wave 2: Asset Integrity & Navigation
- [ ] Task 3: **Finalize Asset Localization Policy**. Verify that all current lesson images are hosted locally and that no external hotlinks remain (from v1.0 feedback).
- [ ] Task 4: **Update Navigation Config**. Add the "Grammatik Themen" page to the VitePress sidebar or top navigation.

## Tasks

### Wave 1: Core Platform Enhancements

<task id="2-1">
<read_first>
- docs/.vitepress/theme/custom.css
</read_first>
<action>
Add CSS counters to `docs/.vitepress/theme/custom.css`:
1. Initialize counter: `.vp-doc { counter-reset: h2-counter; }`
2. Increment and display h2: `h2 { counter-increment: h2-counter; counter-reset: h3-counter; }` and `h2::before { content: counter(h2-counter) ". "; }`
3. Increment and display h3: `h3 { counter-increment: h3-counter; }` and `h3::before { content: counter(h2-counter) "." counter(h3-counter) " "; }`
Ensure the numbering only applies to the main content area and not sidebar/nav.
</action>
<acceptance_criteria>
- `custom.css` contains `counter-reset` and `counter-increment` rules.
- View a lesson (e.g., Lektion 03) in the browser; `h2` and `h3` headers must automatically show "X." and "X.Y" prefixes respectively.
</acceptance_criteria>
</task>

## must_haves

- Automatic numbering must be dynamic (CSS-based), not hardcoded in Markdown.
- All media must be server-independent (local).
- Navigation must provide a way to find topics conceptually (Index-Page).
